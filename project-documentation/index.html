Abstract

In this report we will describe the application functionality of creating a Web System for Managing Water Turbines, explain choices of implementation and general purposes for such an application.

Introduction

The report aims to describe the uses of a web system for Managing Water Turbines, along with how did our team go about implementing such a web application.

In today's time, many applications have a web interface, since it's easy to use for both clients and developers who need to find data about a specific topic. Unlike a desktop application for example, a web application is more easily customizable and managable.

The main audiance for this report are people who are interested in web applications for managing different types of real life systems, as well as people who are interested in the concrete applications of a System for Managing Water Turbines.

The purpose of this application is to assist in visualising, comparing and adding different water turbines around the world. The main benefit of such a web system is, of course, the user-friendliness that it provides for the specific tassk. Our objectives in developing this app are to help people interested in such a system have a better experience for managing and visualising Water Turbines.

Overall Description

This product is a replacement for a general system for Managing Water Turbines, with a web interface such that is it easy to use and easily customizable.

Product Functions

  • Login Page
    • Login: provides a way for the user to login to the application if credentials are correct and redirects the user to the home page
    • redirects for pages to create account, reset password
  • Register Page
    • Register: provides a way for the user to create an account and then redirects the user to the login page
    • redirects for pages to login
  • Forgot-Password Page & Reset Password Page
    • Forgot Password: gives the user a way to input their email address, and the server sends a mail to the specific email address with a unique and exipiring link to reset the password
    • Reset Password: can only be accessed through the unique link sent to the email, provides the user with a way to input a new password for their account
  • Home Page
    • Home: provides a way for the user to view the list of all the water turbines in the database
    • has a menu bar to the right that redirects to different pages (Map Page, My HidroPower Page) and a search bar for hidroplants
    • has a top bar that provides redirects to an About Page, A Statistics Page and the User Page
  • Map Page
    • Map: using Leaflet, provides a map for viewing the hidroplants based on their latitude and longitude, and also provides a search bar in order to find a certain hidroplant's coordinates
  • My HidroPower Page
    • My HidroPower: provides a way for the user to input different data about a hidroplant (including a latitude and a longitude), and redirects the user to a page with Live Statistics about the hidroplant, depending on the weather, which the user can change from the interface
  • HidroPower Page
    • HidroPower: when a user clicks on a hidropower from the main page, it redirects the user to a page with the information about that hidroplant, as well as redirects to see the Live Statistics and the RSS Feed for that hidroplant
  • Live Statistics Page
    • Live Statistics: using PlotLy, provides a way for the user to see the live statistics of a hidroplant, depending on the weather, which the user can change from the interface
  • About Page
    • About: provides a way for the user to see the information about certain trivia facts, like International Water Day
  • Statistics Page
    • Statistics: provides a way for the user to see the information about the top 10 hidroplants sorted by their electrical capacity, and, using Chart.js, models graphs to see the data
  • User Page
    • User: provides a way for the user to change the email of their account, to delete their account and to disconnect
  • Admin Specific Page
    • The admin is a user with a special email address that is used to login to the application. The admin can add and delete hidroplants to the database from the home page.

Operating environment and design constraints

This web application can be used in any operation system (Windows, Mac, Linux) that supports one of the main web browsers (like Google Chrome, Firefox, Microsoft Edge). Some design constraints will appear because of the different ways different browsers interpret HTML, CSS, JS but the general functionalities should be the same. The backend server is written in NodeJS, and the version requires to run it is ^8.15.1.

External Interface Requirements

User Interfaces

Our web application provides diffenet sub-pages for the user management system and the hydropower plant management system. For the user management system, there will be pages for choosing between signup/login, and pages for the sign up and log in systems. For the hydropower plant management system, there are: a page for getting information about the top 10 biggest capacity hydropower plants around the world, a page for getting the list of hydropower plants around the world, a page for querying for a specific hydropower plant and visialising details about it and a page for users to define their own hydropower plants. Also there are pages for the User functions (like change email, delete account, disconnect), an about page with generic trivia and a page which provides a RSS Feed.

Backend Interfaces

For the backend, we use an MVC model, models being for the users and hidroplants data, views being for the routes which are exposed to the frontend, and controllers being for implementation of the code which gets the data, parses it if needed, and sends the response back for a specific request.

Communication Interfaces

These interfaces will manage the communication between frontend and backend, managing different HTTP requests (GET, POST, DELETE, PUT, PATCH etc.). Certain routes (all but Login, Regiser, Forgot-Password) are protected using JWT tokens, so the server can assure that the user is logged in. The server exposes the routes, and, from the frontend we make requests using `axios` in order to get the needed data.

System Features

User Management

This application will be able to manage users. Such users can sign in with a new account (username, password, email), in which case the system will add that account (with the password hashed) to the user table in the embedded database, and users can also log in with an existing account (requiring username or email and password to be valid), in which case the database will be queryed to find if such an account exists.

Biggest Water Turbines Visualization

The system will import data about Water Turbines around the world, and users that are logged in will be able to visualize the biggest water turbines existing in the database. Users will be able to see turbine's location, electric power generated, capacity, frequency of rotation, rated speed, average energy production and the turbines used with Fracis type 4 x 56,5 MW vertical axis.

Seach for a specific Turbine

Users will be able to search for a water turbine's name, and if it exists in the database, information about it (mentioned earlier) will be shown, along with it's status (in use, defective etc.), and a live page progress.

Templates for user-defined Water Turbines

Users are also be able to define their own template Water Turbines from My Hidroplant Page, feature which will help project the ideas and visialise how their defined hydropower plant will function, depending of the given specifications.

Other Nonfunctional Requirements

  • The webpage should load fast.
  • The code shall respect the HTML, CSS and JavaScript standards.
  • The system's database for water turbines should be large enough.
  • The system should provide a REST API.
  • Any exception situation should be reported by email.
